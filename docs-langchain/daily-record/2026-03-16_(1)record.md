## 학습

### Opik이란?

LLM 애플리케이션의 실행 흐름(트레이스)을 수집·시각화하는 관측 도구.
LangChain / LangGraph와 통합하여 에이전트의 각 노드 실행, LLM 호출, 툴 호출 등을 대시보드에서 확인할 수 있다.

### LangGraph 연동 방법 2가지

| 방법 | 설명 | 특징 |
|---|---|---|
| **방법 1: `track_langgraph`** | 그래프를 한 번 래핑 | 이후 모든 호출 자동 트래킹, 콜백 불필요 |
| **방법 2: 콜백 직접 전달** | `invoke()` 호출마다 `callbacks=[opik_tracer]` 전달 | 호출마다 다른 트레이서를 쓰고 싶을 때 사용 |

→ **방법 1** 채택: 한 번만 래핑하면 돼서 코드 변경이 최소화됨.

### `OpikTracer` 파라미터

```python
OpikTracer(
    tags=["stock-agent"],            # 대시보드 필터링용 라벨
    metadata={"model": "gpt-4.1"},   # 트레이스 상세에 표시되는 추가 정보
)
```

- `tags`: 특정 에이전트/환경의 트레이스만 모아보기 위한 라벨
- `metadata`: 모델명, 버전 등 트레이스마다 기록하고 싶은 key-value 정보

### `agent_service.py` 변경 상세

**① 모듈 최상단: `_configure_opik()` 정의 및 즉시 호출**

```python
def _configure_opik():
    """settings.OPIK 값을 기반으로 Opik 환경변수를 설정합니다."""
    from app.core.config import settings
    if settings.OPIK is None:
        return
    if settings.OPIK.URL_OVERRIDE:
        os.environ["OPIK_URL_OVERRIDE"] = settings.OPIK.URL_OVERRIDE
    if settings.OPIK.API_KEY:
        os.environ["OPIK_API_KEY"] = settings.OPIK.API_KEY
    if settings.OPIK.WORKSPACE:
        os.environ["OPIK_WORKSPACE"] = settings.OPIK.WORKSPACE
    if settings.OPIK.PROJECT:
        os.environ["OPIK_PROJECT_NAME"] = settings.OPIK.PROJECT

_configure_opik()
```

**② `__init__()`: LLM 초기화 직후 OpikTracer 초기화**

```python
from opik.integrations.langchain import OpikTracer
self.opik_tracer = OpikTracer(
    tags=["stock-agent"],
    metadata={"model": settings.OPENAI_MODEL},
)
```

**③ `_create_agent()`: agent 생성 직후 `track_langgraph` 래핑**

```python
if self.opik_tracer is not None:
    from opik.integrations.langchain import track_langgraph
    self.agent = track_langgraph(self.agent, self.opik_tracer)
```

## 트러블슈팅

### 1. `No module named 'opik.integrations.langgraph'`

**증상**
```
ModuleNotFoundError: No module named 'opik.integrations.langgraph'
```

**원인**
공식 문서의 import 경로를 잘못 분리하여 작성.
`track_langgraph`는 별도 `langgraph` 모듈이 아니라 `langchain` 모듈 안에 있음.

**해결**
```python
# 변경 전 (잘못된 경로)
from opik.integrations.langgraph import track_langgraph

# 변경 후 (올바른 경로)
from opik.integrations.langchain import track_langgraph
```

---

### 2. Opik 405 Not Allowed

**증상**
```
OPIK: Failed to process CreateSpansBatchMessage. Error: status_code: 405
OPIK: Failed to process CreateTraceBatchMessage. Error: status_code: 405
```

**원인**
`OPIK_URL_OVERRIDE`에 base URL만 설정했을 때 Opik SDK가 nginx 프론트엔드로 API 요청을 보내 405 반환.
self-hosted Opik의 API 경로는 base URL에 `/api`를 붙여야 함.

**해결**
```
# 변경 전
OPIK__URL_OVERRIDE=https://opik-edu.didim365.app

# 변경 후
OPIK__URL_OVERRIDE=https://opik-edu.didim365.app/api
```
