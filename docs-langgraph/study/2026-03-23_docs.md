## LangGraph란?

LangGraph는 **복잡한 AI 에이전트를 안정적으로 설계하기 위한 저수준 오케스트레이션 프레임워크**입니다.

Klarna, Replit, Elastic 등 여러 기업들이 신뢰하며 사용하고 있으며, **장기 실행(long-running)** 및 **상태 유지(stateful)** 에이전트를 구축·관리·배포하기 위한 런타임을 제공합니다.

### LangChain과의 관계

- LangGraph는 **LangChain 없이도 독립적으로 사용** 가능합니다.
- LangChain은 모델·툴 통합에 편리한 컴포넌트를 제공하며, LangGraph와 함께 사용하면 시너지를 냅니다.
- 에이전트를 처음 만든다면 LangChain의 사전 빌드된 에이전트 아키텍처로 시작해도 좋습니다.
- LangGraph는 **프롬프트나 아키텍처를 추상화하지 않으며**, 오케스트레이션 인프라에 집중합니다.

## Hello World 예제

LangGraph의 기본 동작을 이해하는 가장 간단한 예제입니다.

```python
from langgraph.graph import StateGraph, MessagesState, START, END

def mock_llm(state: MessagesState):
    return {"messages": [{"role": "ai", "content": "hello world"}]}

graph = StateGraph(MessagesState)
graph.add_node(mock_llm)
graph.add_edge(START, "mock_llm")
graph.add_edge("mock_llm", END)
graph = graph.compile()

graph.invoke({"messages": [{"role": "user", "content": "hi!"}]})
```

**핵심 흐름:**
1. `StateGraph` 로 그래프 정의
2. 노드(함수) 추가
3. 엣지(흐름)로 노드 연결
4. `compile()` 로 그래프 완성
5. `invoke()` 로 실행

## LangGraph의 핵심 장점

| 기능 | 설명 |
|---|---|
| **Durable Execution (내구성 있는 실행)** | 장애 발생 시에도 마지막 지점에서 재개 가능. 장시간 실행 워크플로우 지원 |
| **Human-in-the-loop** | 언제든지 에이전트 상태를 검사하고 수정하여 인간의 감독을 통합 가능 |
| **Comprehensive Memory (종합 메모리)** | 단기 작업 메모리(현재 대화)와 세션 간 장기 메모리 모두 지원 |
| **LangSmith 디버깅** | 실행 경로 추적, 상태 전환 캡처, 런타임 지표 제공으로 복잡한 에이전트 동작 가시화 |
| **Production-ready 배포** | 상태 유지형 장기 실행 워크플로우를 위한 확장 가능한 인프라로 안정적 배포 |

## LangGraph 생태계

```
┌─────────────────────────────────────────────────────────────┐
│                      LangChain 생태계                        │
│                                                             │
│  ┌─────────────┐   ┌──────────────┐   ┌─────────────────┐  │
│  │  LangGraph  │──▶│  LangSmith   │   │   LangChain     │  │
│  │ (오케스트레이션) │   │ (관찰/평가)    │   │ (통합/컴포넌트)   │  │
│  └─────────────┘   └──────────────┘   └─────────────────┘  │
│         │                  │                   │            │
│         └──────────────────┴───────────────────┘            │
│                      함께 사용 가능 (각각 독립적)               │
└─────────────────────────────────────────────────────────────┘
```

- **LangSmith 관찰성**: 요청 추적, 출력 평가, 배포 모니터링을 한 곳에서
- **LangSmith 배포**: 장기 실행 상태 유지 워크플로우용 배포 플랫폼. Studio에서 시각적 프로토타이핑 지원
- **LangChain**: LLM 애플리케이션 개발을 위한 통합 컴포넌트 및 LangGraph 기반의 에이전트 추상화 제공

## LangGraph의 기술적 영감

LangGraph는 다음 프로젝트들에서 영감을 받아 설계되었습니다:

- **[Google Pregel](https://research.google/pubs/pub37252/)**: 그래프 처리 시스템 (슈퍼스텝 기반 실행 모델)
- **[Apache Beam](https://beam.apache.org/)**: 분산 데이터 처리 프레임워크
- **[NetworkX](https://networkx.org/)**: 공개 인터페이스 설계에서 영감

## LangGraph의 두 가지 API

LangGraph는 에이전트를 정의하는 두 가지 방법을 제공합니다:

### Graph API (그래프 API)
- 노드와 엣지로 에이전트를 **명시적으로 구조화**
- 복잡한 조건 분기, 병렬 처리, 시각화에 적합
- 상태(State)를 TypedDict로 명시적으로 정의

### Functional API (함수형 API)
- 기존 Python 코드에 **최소한의 변경**만으로 LangGraph 기능 추가
- `if/else`, `for` 루프 등 표준 Python 제어 흐름 사용
- 빠른 프로토타이핑, 선형 워크플로우에 적합

> 두 API는 **같은 런타임**을 공유하며, 하나의 애플리케이션에서 함께 사용 가능합니다.
